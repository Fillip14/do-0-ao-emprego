1. O que é Git e para que serve um commit? Git é um serviço no qual podemos armazenar edições dos nossos scripts por meio dos commits, que são comentários que colocamos nas edições que fazemos. Exemplo mudança de nome de variavel seria um commit para salvar no github.

2. O que acontece por baixo dos panos quando você digita um site no navegador e aperta Enter? O endereço no qual foi digitado vai para um servidor no qual estão os dados do site como html, css, ou react, e esses dados são transferidos para o navegador, o navegador interpreta tudo isso e exibe na tela. Requisições e respostas também podem ocorrer, cookies, dados armazenados.

3. O que é uma API? API é um "conector" entre duas coisas, exemplo para um servidor conversar com um banco de dados é preciso de uma API, nela temos os comandos necessário para nos conectarmos com o banco, como se o servidor falasse uma linguagem e o banco falasse outra, essa API intermedia e faz com que os dois se "entendam".

4. Qual a diferença entre front-end e back-end? Front-end é a interface na qual o usuário está vendo, é tudo que está na "frente", o site em sim. Back-end é o que o usuário não ve, é a lógica por trás, o que se comunica com servidor, onde está toda lógica.

5. O que é um banco de dados e por que não guardar tudo em arquivos? Banco de dados como o próprio nome já diz é um lugar no qual guardamos dados de forma segura, SQL ou noSQL, são meios nos quais conseguimos organizar dados que precisam ser salvos. Guardar em arquivos se tornaria um gasto de computação desncessário e extremamente vulnerável.

6. Qual a diferença entre GET e POST? GET, POST, PATCH, DELETE, são formas na quais sao feitas requests em websites. O get eu apenas peço para ele me trazer alguma informação, exemplo pegue o nome de usuário, post eu estou colocando algo, exemplo coloque o nome de usuário no DB.

7. O que significa um status 200, 404 e 500? Status 200 é statuso ok, deu certo. Os 404 e 500 são status de erros mas não sei especificar qual.

8. O que é JSON e por que as APIs o utilizam? JSON é um meio de salvar dados e transmitir dados entre diversas linguagens, databases, é uma forma "universal" de transmitir dados. Quando faço uma request geralmente as reponse são em JSON.
